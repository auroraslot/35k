> 我初次开始做算法题，平时是业务开发人员，为什么我99%的题目都做不出来，都要看题解

这是非常正常的现象，不是你的问题。让我帮你分析一下原因和破局方法。

**为什么做不出来？**

**业务开发和算法题是两种完全不同的能力。**

| 业务开发                | 算法题               |
| ----------------------- | -------------------- |
| 用框架、调API、组装功能 | 从零手写数据结构操作 |
| 知道"用什么"            | 要求"怎么造"         |
| 需求明确，Google能搜到  | 需要识别隐藏模式     |
| 重经验积累  | 重思维训练     |

你写了几年业务代码，并不会让你自动会解算法题，就像一个开了十年车的老司机，不代表能修发动机。**这是两个独立的技能树。**

**做不出来的真正原因**

**不是智力问题，是没见过"套路"。**

算法题本质上是 **模式匹配**：
- 看到"有序数组" → 想到二分/双指针
- 看到"连续子数组" → 想到滑动窗口/前缀和
- 看到"所有可能" → 想到回溯

你做不出来，是因为脑子里还没有这套"模式库"。看题解不是丢人，**看题解就是在建立模式库的过程**。

**正确的刷题方法**

**阶段一：抄（前50题）**

没错，就是**抄题解**，但要这样抄：

1. 先自己想 **10-15 分钟**，想不出来就看题解

2. **理解**题解的每一行为什么这么写

3. 关掉题解，**自己默写**一遍
3. 第二天，**再默写**一遍

> 这个阶段的目标不是"做出来"，而是积累模式。



**阶段二：仿（50-150题）**

见到新题时开始能想到："这题好像和之前某道题类似"

1. 先想 **20-25 分钟**
2. 想不出来时，只看**提示**（用什么算法），不看完整题解
3. 知道方向后自己写



阶段三：创（150题以后）

大部分 Medium 能独立做出来，开始挑战 Hard。

**具体建议**

**按专题刷，不要随机刷。** 一个专题集中做 5-8 道：

建议顺序：
1. 双指针（你刚学的，继续）  ← 现在在这里
2. 哈希表
3. 滑动窗口
4. 二分查找
5. BFS/DFS + 二叉树
6. 回溯
7. 动态规划（最难，放后面）

每个专题做完后你会发现：**同一类题的代码结构几乎一样**，变的只是细节。这就是"套路"。

**一句话总结**

> 看题解不丢人，看了不总结才亏。 你现在缺的不是天赋，是模式库。刷够 100 题，质变就会来。
